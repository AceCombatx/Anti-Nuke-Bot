Quick Setup Guide

Get Anti Nuke Bot V5 protecting your server in under 5 minutes

1

Invite the Bot

Add Anti Nuke Bot V5 to your server with Administrator permissions

Invite Bot
2

Run Setup Wizard

Use the interactive setup wizard for guided configuration. This command will walk you through essential settings.

/setup-wizard
3

Configure Logging

Set up a secure logging channel for all protection events. This is required for ALL protection features to work, as it provides critical audit trails.

/setup-logging #security-logs
4

Add Trusted Staff

Whitelist your administrators to prevent false positives. Only whitelisted users can make admin changes without triggering protection measures.

/whitelist add @Admin
5

Enable Protection

Activate all protection modules with recommended settings. You can fine-tune these later using their respective configuration commands.

/antinuke-config enabled:true /antiraid-config enabled:true /antispam-config enabled:true /antiselfbot-config enabled:true /antialt-config enabled:true
6

Verify Setup

Check that all protection features are active and properly configured

/protection-status
7

Create Backup

Save your configuration for easy restoration or transfer. This sends you a JSON file with all your settings. Store safely.

/backup action:create

Pro Tip!

Use the /setup-wizard command for an interactive setup experience that will guide you through all configuration options with recommendations based on your server size and needs.

For troubleshooting, use /errors to access our comprehensive problem-solving guide.

Complete Command Reference

All commands available in Anti Nuke Bot V5

Setup & Configuration

/setup-wizard
Interactive setup wizard for complete bot configuration. Guides you through setting up logging, protection levels, whitelisting administrators, and enabling protection modules.
/setup-logging
Configure the dedicated logging channel for all protection events. This is crucial for monitoring and auditing bot actions.
**Arguments:** * `channel`: The text channel to send logs to.
/whitelist
Manage trusted users who can bypass protection and configure the bot. Whitelisting is essential to prevent false positives.
**Arguments:** * `action`: `add-user`, `remove-user`, `add-role`, `remove-role`, or `list`. * `user` (optional): The user to add or remove. * `role` (optional): The role to add or remove.
/protection-status
View a comprehensive overview of all protection features and their current status. This command helps you ensure that all systems are active and properly configured.

Protection Configuration

/antinuke-config
Configure anti-nuke protection settings, including thresholds, punishments, and auto-restore. This command allows you to customize the bot's response to potential server nuking attempts.
**Arguments:** * `enabled`: `true` or `false` to enable or disable the module. * `channel_delete_threshold`: Number of channel deletions to trigger action. * `role_delete_threshold`: Number of role deletions to trigger action. * `channel_create_threshold`: Number of channel creations to trigger action. * `role_create_threshold`: Number of role creations to trigger action. * `name_change_threshold`: Number of name changes to trigger action. * `punishment`: `ban` or `kick` to specify the punishment for detected nukers. * `auto_restore`: `True` or `False` to enable or disable automatic restoration of deleted items. * `webhook_protection`: `True` or `False` to enable or disable webhook protection. * `invite_protection`: `True` or `False` to enable or disable invite protection. * `emoji_protection`: `True` or `False` to enable or disable emoji protection. * `sticker_protection`: `True` or `False` to enable or disable sticker protection. * `thread_protection`: `True` or `False` to enable or disable thread protection. * `server_settings_protection`: `True` or `False` to enable or disable server settings protection.
/antiraid-config
Configure anti-raid protection settings, such as join rate, time window, and punishment. This command helps you manage and mitigate potential raids on your server.
**Arguments:** * `enabled`: `true` or `false` to enable or disable the module. * `join_rate`: Number of joins within the time window to trigger raid mode. * `time_window`: Time window in seconds to monitor join rate. * `punishment`: `ban` or `kick` to specify the punishment for detected raiders. * `auto_lockdown`: `True` or `False` to enable or disable automatic lockdown during raids. * `verification_boost`: `True` or `False` to enable or disable verification level boost during raids. * `new_account_threshold`: Days to consider an account 'new'.
/antispam-config
Configure anti-spam protection, including message thresholds, time windows, and duplicate content detection. This command allows you to customize the bot's response to spamming behavior.
**Arguments:** * `enabled`: `true` or `false` to enable or disable the module. * `message_threshold`: Number of messages within the time window to trigger action. * `time_window`: Time window in seconds to monitor message rate. * `punishment`: `ban`, `kick`, or `mute` to specify the punishment for detected spammers. * `mute_duration`: Duration of mute in minutes (if punishment is mute). * `duplicate_detection`: `True` or `False` to enable or disable duplicate message detection. * `link_protection`: `True` or `False` to enable or disable suspicious link protection. * `caps_protection`: `True` or `False` to enable or disable excessive caps protection. * `emoji_spam_protection`: `True` or `False` to enable or disable emoji spam protection.
/antiselfbot-config
Configure selfbot detection settings, including sensitivity, risk thresholds, and punishment. This command allows you to customize the bot's response to potential selfbot activity.
**Arguments:** * `enabled`: `true` or `false` to enable or disable the module. * `sensitivity`: `low`, `medium`, or `high` to specify the detection sensitivity level. * `punishment`: `ban` or `kick` to specify the punishment for detected selfbots.
/antialt-config
Configure alt account detection settings, such as minimum account age and punishment. This command helps you manage and mitigate potential alt accounts joining your server.
**Arguments:** * `enabled`: `true` or `false` to enable or disable the module. * `min_account_age`: Minimum account age in days to allow joining. * `punishment`: `ban` or `kick` to specify the punishment for detected alt accounts. * `whitelist_verification`: `True` or `False` to allow verification for borderline cases instead of immediate punishment.
/messagefilter-config
Configure advanced message filtering rules, including custom word lists and link protection. This command allows you to customize the bot's response to unwanted content.
**Arguments:** * `enabled`: `true` or `false` to enable or disable the module. * `punishment`: `delete`, `mute`, `kick`, or `ban` to specify the punishment for detected violations. * invite_filter: `True` or `False` to enable or disable invite filtering. * link_filter: `True` or `False` to enable or disable link filtering. * add_bad_word: Word to add to the bad words list. * remove_bad_word: Word to remove from the bad words list. * list_bad_words: `True` to show the current bad words list.
/antimention-config
Configure mass mention protection settings, including mention thresholds and punishment. This command helps you manage and mitigate potential mass mention spam.
**Arguments:** * `enabled`: `true` or `false` to enable or disable the module. * `mention_threshold`: Number of mentions to trigger action. * `punishment`: `ban`, `kick`, or `mute` to specify the punishment for detected mass mentioners. * `mute_duration`: Duration of mute in minutes (if punishment is mute). * `role_mention_protection`: `True` or `False` to enable or disable role mention protection. * `everyone_protection`: `True` or `False` to enable or disable @everyone/@here protection.

Emergency Response

/raid-mode
Manually activate or deactivate raid protection mode for immediate server control. This command is useful during active raids.
**Arguments:** * `action`: `activate` or `deactivate` to specify the action to perform. * `reason` (optional): Reason for the action.
/lockdown
Initiate an emergency server lockdown, instantly protecting all channels. This command is useful during severe attacks.
**Arguments:** * `action`: `enable` or `disable` to specify the action to perform. * `reason` (optional): Reason for the action.

Configuration Management

/backup action:create
Create a comprehensive JSON backup file of all bot settings and configurations. This command is essential for saving your current setup.
**Arguments:** * `action`: Must be set to `create`.
/backup action:restore
Restore bot configuration from an uploaded backup JSON file. This command allows you to revert to a previous configuration.
**Arguments:** * `action`: Must be set to `restore`.
/backup action:toggle_auto
Enable or disable automatic daily backups for your server. This command ensures that your settings are regularly backed up.
**Arguments:** * `action`: Must be set to `toggle_auto`.
/backup action:status
View the current backup status, including the last backup timestamp and auto-backup setting. This command provides information about your backup configuration.
**Arguments:** * `action`: Must be set to `status`.
/backup action:delete_data
Permanently delete all backup data stored for your server. This command is useful for removing sensitive information.
**Arguments:** * `action`: Must be set to `delete_data`.
/backup action:reset_triggered
Reset the protection triggered status, allowing automatic backups to resume. This command is useful after a false positive.
**Arguments:** * `action`: Must be set to `reset_triggered`.

Common questions about Anti Nuke Bot V5

  • Expanded Anti-Nuke: Now includes Webhook, Invite, Emoji, and Sticker protection.
  • Trusted Bots Whitelist: Prevents false positives from other legitimate bots.

How accurate is the selfbot detection?

Our selfbot detection system uses advanced behavioral analysis with multiple detection factors like message patterns, response speed, and activity patterns. The accuracy depends on the sensitivity level you configure:

  • Low Sensitivity: Fewer false positives, may miss sophisticated selfbots. Good for servers with very active users.
  • Medium Sensitivity (Recommended): Balanced detection with a reasonable false positive rate. Ideal for most servers.
  • High Sensitivity: Catches more selfbots but has a higher false positive rate, requiring more manual review.

We recommend starting with medium sensitivity and adjusting based on your server's needs. Always manually verify detections before taking permanent action.

Why aren't the protection features working?

All protection features require proper setup and permissions before activation. Use our interactive troubleshooting tools:

  1. Run /setup-wizard for guided configuration.
  2. Ensure logging is set up: /setup-logging #channel. This is critical for all features.
  3. Add trusted admins: /whitelist add @Admin. Non-whitelisted admins will trigger protection.
  4. Enable features individually with their config commands (e.g., /antinuke-config enabled:true).
  5. Verify the bot has all required permissions and its role is at the very top of your server's role list.
  6. Check overall setup status: /protection-status
  7. For specific issues, use /errors

What permissions does the bot need?

For optimal protection and functionality, Anti Nuke Bot V5 highly recommends and ideally requires **Administrator** permissions. This ensures:

  • Complete access to monitor all server activities and audit logs.
  • Ability to reverse malicious actions instantly (e.g., deleting channels, recreating roles).
  • Permission to manage roles, channels, members (kick, ban, timeout), and webhooks.
  • No permission conflicts that could compromise protection.

Critical Importance of Role Position: The bot's role **must be positioned at the very top** of your server's role hierarchy (just below the server owner) to effectively manage and protect all other roles and members.

You can use the in-bot command /errors and select "Check Permissions" to get a detailed report on missing permissions.

How do I prevent false positives?

To minimize false positives and ensure legitimate actions are not flagged:

  • Whitelist trusted staff: Use /whitelist add @User for all administrators and moderators. The server owner is automatically whitelisted.
  • Adjust sensitivity: For features like Anti-Selfbot, start with lower sensitivity (e.g., /antiselfbot-config sensitivity:low) and increase gradually as needed.
  • Configure thresholds: Set appropriate limits for your server size and activity level. For example, increase message_threshold for Anti-Spam or join_rate for Anti-Raid.
  • Use warning mode: Some features allow a "warn" punishment first, giving you time to manually review before implementing harsher actions.
  • Monitor logs: Regularly review detection logs in your configured logging channel to identify patterns and adjust settings.
  • Regular review: Periodically use the respective -config commands or /protection-status to review and adjust settings.

Can I customize the detection algorithms?

Yes! Anti Nuke Bot V5 offers extensive customization options for its detection algorithms and responses:

  • Sensitivity Levels: Adjust detection sensitivity for each module (e.g., low/medium/high for Anti-Selfbot).
  • Custom Thresholds: Set specific limits for message rates, join rates, channel/role deletions, and more, tailored to your server's needs.
  • Punishment Escalation: Configure progressive punishment systems, from warnings and mutes to kicks and bans.
  • Time Windows: Adjust monitoring periods for different activities to catch rapid bursts or slower, coordinated attacks.
  • Whitelist Rules: Create complex whitelist configurations for users, roles, or even specific channels.
  • Custom Filters: Add server-specific content filters for messages, including custom words and regex patterns.

Use the respective -config commands (e.g., /antinuke-config, /antispam-config) or the interactive /setup-wizard for guided customization.
